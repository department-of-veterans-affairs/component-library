# Architecture Decision Records

This section contains Architecture Decision Records (ADRs) that document significant technical decisions made by the team.

## What is an ADR?

An Architecture Decision Record captures:
- **Context**: The situation and factors that led to the decision
- **Decision**: What was decided
- **Consequences**: The resulting effects, both positive and negative
- **Alternatives**: Other options that were considered

## Creating a New ADR

Click **Edit Content** in the top navigation or go directly to [/admin/#/collections/adr](/admin/#/collections/adr) to create a new ADR.

## ADR Status

- **Proposed** - Under discussion
- **Accepted** - Decision has been made
- **Deprecated** - No longer relevant
- **Superseded** - Replaced by a newer decision
